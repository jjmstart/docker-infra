# 操作手册：v1.1 → v1.2

## 演进概述

将 MySQL 由 gz-03 单点主库升级为**一主两从 + 半同步复制 + GTID + ProxySQL 读写分离 + bj-01 延迟复制**，并接入 mysqld_exporter 监控，消除 MySQL 单点故障风险，实现读写分离与操作级灾备。

---

## 前置条件

| 项目 | 要求 |
|------|------|
| 当前架构版本 | V1.1（参见 `architecture/v1.1.md`） |
| gz-03 MySQL | 单点主库运行中，无 binlog/GTID；compose 路径 `/opt/docker/backend/mysql/docker-compose.yml` |
| gz-02 | 无 MySQL 实例；磁盘空间 ≥ gz-03 数据目录大小 + 20% |
| bj-01 | 无 MySQL 实例；磁盘空间同上 |
| gz-03 → gz-02 | Tailscale 互通（~5ms） |
| gz-03 → bj-01 | Tailscale 互通（~35ms） |
| 业务停机窗口 | Phase 1 主库重启约需 30s 停机；可在业务低峰期操作 |

## 各 Phase 详细步骤

### Phase 1：主库准备（gz-03）

**① 目标说明**

开启 binlog、GTID、半同步主端插件，创建复制专用账号 `repl_user`。此阶段需重启 MySQL 容器（约 30s 停机），建议业务低峰执行。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

编辑 `/opt/docker/backend/mysql/docker-compose.yml`，在 `command` 块中新增以下参数：

```yaml
command:
  - --log-bin=mysql-bin
  - --server-id=1
  - --gtid_mode=ON
  - --enforce_gtid_consistency=ON
  - --binlog_format=ROW
  - --binlog_expire_logs_seconds=604800
  - --plugin-load-add=rpl_semi_sync_source=semisync_source.so
  - --rpl_semi_sync_source_enabled=ON
  - --rpl_semi_sync_source_wait_for_replica_count=1
  - --rpl_semi_sync_source_timeout=5000
```

重启主库容器：

```bash
cd /opt/docker/backend/mysql
docker compose up -d --force-recreate
```

创建复制账号（`repl_user` 密码存于 `/opt/docker/backend/mysql/.env` → `MYSQL_REPL_PASSWORD`）：

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
CREATE USER IF NOT EXISTS 'repl_user'@'%' IDENTIFIED BY 'your_repl_password';
GRANT REPLICATION SLAVE ON *.* TO 'repl_user'@'%';
FLUSH PRIVILEGES;
EOF
```

> **注意**：`docker exec` 与 heredoc 一起使用时必须用 `-i`，不能加 `-t`，否则会报 `the input device is not a TTY` 错误。

**④ 预期输出**

```
Recreating mysql ... done
```

**⑤ 验证方法**

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "
SHOW MASTER STATUS;
SHOW VARIABLES LIKE 'gtid_mode';
SHOW VARIABLES LIKE 'rpl_semi_sync%';
"
```

期望：
- `SHOW MASTER STATUS` 有 File 和 Position 输出
- `gtid_mode = ON`
- `rpl_semi_sync_source_enabled = ON`（此时无从库，status 会显示 OFF，属正常现象，等 Phase 2 后变为 ON）

**⑥ 回滚方式**

去掉 `command` 中新增的参数，执行 `docker compose up -d --force-recreate`，从库尚未建立，主库可独立服务。

### Phase 2：同城从库（gz-02）

**① 目标说明**

在 gz-02 部署 MySQL Slave1（server-id=2），开启半同步从端，从 gz-03 快照初始化，建立实时主从复制。

**② 节点**：gz-03（快照）→ gz-02（部署）

**③ 具体命令**

**步骤 2.1：gz-03 导出快照**

```bash
# 在 gz-03 执行
docker exec -i mysql mysqldump \
  -uroot -p"${MYSQL_ROOT_PASSWORD}" \
  --single-transaction \
  --master-data=2 \
  --set-gtid-purged=ON \
  --all-databases \
  > /tmp/mysql_snapshot.sql

echo "快照大小：$(du -sh /tmp/mysql_snapshot.sql)"
```

**步骤 2.2：传输快照至 gz-02（经 Tailscale，不走公网）**

```bash
# 在 gz-03 执行
scp /tmp/mysql_snapshot.sql root@100.79.132.125:/tmp/mysql_snapshot.sql
```

**步骤 2.3：gz-02 创建 MySQL Slave1 compose**

在 gz-02 创建 `/opt/docker/backend/mysql/docker-compose.yml`，`server-id=2`，`command` 中加入：

```yaml
command:
  - --server-id=2
  - --read_only=ON
  - --gtid_mode=ON
  - --enforce_gtid_consistency=ON
  - --binlog_format=ROW
  - --plugin-load-add=rpl_semi_sync_replica=semisync_replica.so
  - --rpl_semi_sync_replica_enabled=ON
```

```bash
cd /opt/docker/backend/mysql
docker compose up -d
```

**步骤 2.4：导入快照**

```bash
# 在 gz-02 执行
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" < /tmp/mysql_snapshot.sql
```

**步骤 2.5：建立复制**

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='100.92.5.116',
  SOURCE_PORT=3306,
  SOURCE_USER='repl_user',
  SOURCE_PASSWORD='your_repl_password',
  SOURCE_AUTO_POSITION=1;
START REPLICA;
EOF
```

**④ 预期输出**

导入快照无错误输出；`START REPLICA` 无错误。

**⑤ 验证方法**

```bash
# 在 gz-02 执行
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "SHOW REPLICA STATUS\G" | grep -E "Seconds_Behind_Source|Replica_IO_Running|Replica_SQL_Running"
# 期望：
# Replica_IO_Running: Yes
# Replica_SQL_Running: Yes
# Seconds_Behind_Source: 0

# 在 gz-03 验证半同步已激活（有从库 ACK 后从 OFF 变 ON）
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "SHOW STATUS LIKE 'rpl_semi_sync_source_status';"
# 期望：rpl_semi_sync_source_status = ON
```

**⑥ 回滚方式**

```bash
# gz-02：停止复制并删除容器（不影响 gz-03 主库）
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "STOP REPLICA; RESET REPLICA ALL;"
cd /opt/docker/backend/mysql && docker compose down
```

### Phase 3：异地从库（bj-01）

**① 目标说明**

在 bj-01 部署 MySQL Slave2（server-id=3），开启半同步从端，并配置 `SOURCE_DELAY=3600`（1 小时延迟复制），作为异地灾备防误删节点。

**② 节点**：gz-03（快照复用）→ bj-01（部署）

**③ 具体命令**

**步骤 3.1：传输快照至 bj-01**（可复用 Phase 2 的快照，若数据无变化）

```bash
# 在 gz-03 执行
scp /tmp/mysql_snapshot.sql root@100.118.69.78:/tmp/mysql_snapshot.sql
```

若时间间隔较长，需重新导出快照（同 Phase 2 步骤 2.1）。

**步骤 3.2：bj-01 创建 MySQL Slave2 compose**

在 bj-01 创建 `/opt/docker/backend/mysql/docker-compose.yml`，`server-id=3`，`command` 中加入（与 Phase 2 相同，仅 `server-id` 不同）：

```yaml
command:
  - --server-id=3
  - --read_only=ON
  - --gtid_mode=ON
  - --enforce_gtid_consistency=ON
  - --binlog_format=ROW
  - --plugin-load-add=rpl_semi_sync_replica=semisync_replica.so
  - --rpl_semi_sync_replica_enabled=ON
```

```bash
cd /opt/docker/backend/mysql
docker compose up -d
```

**步骤 3.3：导入快照 + 建立复制**

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" < /tmp/mysql_snapshot.sql

docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='100.92.5.116',
  SOURCE_PORT=3306,
  SOURCE_USER='repl_user',
  SOURCE_PASSWORD='your_repl_password',
  SOURCE_AUTO_POSITION=1;
START REPLICA;
EOF
```

**步骤 3.4：配置延迟复制**

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
STOP REPLICA SQL_THREAD;
CHANGE REPLICATION SOURCE TO SOURCE_DELAY = 3600;
START REPLICA SQL_THREAD;
EOF
```

**④ 预期输出**

各步骤无错误输出。

**⑤ 验证方法**

```bash
# 在 bj-01 执行
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "SHOW REPLICA STATUS\G" | grep -E "SQL_Delay|Seconds_Behind_Source|Replica_IO_Running|Replica_SQL_Running"
# 期望：
# Replica_IO_Running: Yes
# Replica_SQL_Running: Yes
# SQL_Delay: 3600
# Seconds_Behind_Source: 约 3600（正常，IO_Thread 实时接收，SQL_Thread 延迟应用）
```

**⑥ 回滚方式**

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "STOP REPLICA; RESET REPLICA ALL;"
cd /opt/docker/backend/mysql && docker compose down
```

### Phase 4：ProxySQL 读写分离（gz-03）

**① 目标说明**

在 gz-03 部署 ProxySQL，配置写 hostgroup 0（Master）/ 读 hostgroup 1（Slave1），修改 ruoyi-admin-1 和 ruoyi-admin-2 的数据源指向 ProxySQL :6033，实现读写分离与应用解耦。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

**步骤 4.1：创建 ProxySQL compose**

创建 `/opt/docker/backend/proxysql/docker-compose.yml`：

```yaml
services:
  proxysql:
    image: proxysql/proxysql:latest
    container_name: proxysql
    volumes:
      - ./proxysql.cnf:/etc/proxysql.cnf
    ports:
      - "100.92.5.116:6033:6033"
      - "100.92.5.116:6032:6032"
    networks:
      - global_gateway
    restart: unless-stopped

networks:
  global_gateway:
    external: true
```

**步骤 4.2：创建 `proxysql.cnf`**

关键配置段：

```
datadir="/var/lib/proxysql"

mysql_servers =
(
  { address="100.92.5.116", port=3306, hostgroup=0, comment="Master" },
  { address="100.79.132.125", port=3306, hostgroup=1, comment="Slave1" }
)

mysql_query_rules =
(
  {
    rule_id=1,
    active=1,
    match_pattern="^SELECT",
    destination_hostgroup=1,
    apply=1
  }
)

mysql_users =
(
  { username="ruoyi_user", password="your_password", default_hostgroup=0 }
)
```

> **注意**：`proxysql.cnf` 中后端节点字段名为 `address`，而通过 ProxySQL Admin SQL 操作时字段名为 `hostname`，二者不一致，勿混淆。

**步骤 4.3：启动 ProxySQL**

```bash
cd /opt/docker/backend/proxysql
docker compose up -d
```

**步骤 4.4：修改 ruoyi 数据源**

修改 ruoyi-admin-1（gz-03）和 ruoyi-admin-2（gz-02）的 application.yml 或环境变量，将 MySQL URL 由原 `100.92.5.116:3306` 改为 `100.92.5.116:6033`，重启服务：

```bash
cd /opt/docker/backend/ruoyi
docker compose up -d --force-recreate
```

gz-02 的 ruoyi-admin-2 同步修改并重启。

**④ 预期输出**

```
Creating proxysql ... done
```

**⑤ 验证方法**

```bash
# 连接 ProxySQL Admin（MySQL 协议，默认密码 admin）
docker exec -i proxysql mysql -h127.0.0.1 -P6032 -uadmin -padmin <<'EOF'
SELECT hostgroup_id, hostname, port, status FROM runtime_mysql_servers;
SELECT * FROM stats_mysql_query_digest LIMIT 5;
EOF
# 期望：hostgroup 0 Master / hostgroup 1 Slave1 均为 ONLINE
# 业务产生读写后，stats_mysql_query_digest 中 SELECT 命中 hostgroup 1，写操作命中 hostgroup 0
```

**⑥ 回滚方式**

```bash
# 将 ruoyi 数据源改回 100.92.5.116:3306，重启 ruoyi 服务
# 停止 ProxySQL（不影响数据库）
cd /opt/docker/backend/proxysql && docker compose down
```

### Phase 5：监控接入（gz-01 + 各节点）

**① 目标说明**

在 gz-03 / gz-02 / bj-01 各部署 `mysqld_exporter` 容器，暴露 Tailscale IP:9104，在 gz-01 Prometheus 增加三个采集 targets，Grafana 导入 MySQL 监控面板。

**② 节点**：gz-03 / gz-02 / bj-01（部署 exporter）+ gz-01（配置采集）

**③ 具体命令**

**各数据库节点**（以 gz-03 为例，gz-02/bj-01 同理，修改 IP）：

在各节点的 `/opt/docker/backend/mysql/docker-compose.yml` 中增加 `mysqld-exporter` 服务：

```yaml
  mysqld-exporter:
    image: prom/mysqld-exporter:latest
    container_name: mysqld-exporter
    environment:
      - DATA_SOURCE_NAME=exporter_user:exporter_password@tcp(mysql:3306)/
    ports:
      - "100.92.5.116:9104:9104"
    networks:
      - global_gateway
    depends_on:
      - mysql
```

同时在 MySQL 中创建 exporter 账号：

```bash
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
CREATE USER IF NOT EXISTS 'exporter_user'@'localhost' IDENTIFIED BY 'exporter_password';
GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter_user'@'localhost';
FLUSH PRIVILEGES;
EOF
```

启动：

```bash
docker compose up -d mysqld-exporter
```

**gz-01：更新 Prometheus 配置**

编辑 `/opt/docker/monitor/prometheus/prometheus.yml`，新增采集配置：

```yaml
  - job_name: 'mysql'
    static_configs:
      - targets:
          - '100.92.5.116:9104'   # gz-03 MySQL Master
          - '100.79.132.125:9104' # gz-02 MySQL Slave1
          - '100.118.69.78:9104'  # bj-01 MySQL Slave2
```

重启 Prometheus：

```bash
cd /opt/docker/monitor
docker compose restart prometheus
```

**Grafana 导入面板**：导入 MySQL Overview（Dashboard ID：7362 或同类）。

**配置告警规则**（`prometheus.yml` 或独立 rules 文件）：

```yaml
groups:
  - name: mysql
    rules:
      - alert: MySQLReplicationLag
        expr: mysql_slave_status_seconds_behind_master{instance!="100.118.69.78:9104"} > 10
        for: 2m
        labels:
          severity: warning
      - alert: MySQLReplicationDown
        expr: mysql_slave_status_slave_io_running == 0 or mysql_slave_status_slave_sql_running == 0
        for: 1m
        labels:
          severity: critical
      - alert: MySQLSemiSyncReplicasZero
        expr: mysql_global_status_rpl_semi_sync_source_clients == 0
        for: 5m
        labels:
          severity: warning
```

**④ 预期输出**

各节点 mysqld-exporter 启动成功；Prometheus UI 中三个 mysql targets 均为 UP。

**⑤ 验证方法**

```bash
# 验证 exporter 指标
curl http://100.92.5.116:9104/metrics | grep mysql_slave_status
# 期望：输出复制状态相关指标

# Prometheus 查询
curl -s 'http://localhost:9090/api/v1/query?query=mysql_up' | python3 -m json.tool
# 期望：三个 mysqld_exporter 实例全部 mysql_up=1
```

**⑥ 回滚方式**

```bash
# 各节点停止 mysqld-exporter
docker stop mysqld-exporter && docker rm mysqld-exporter
# gz-01 还原 prometheus.yml，重启 Prometheus
```

### Phase 6：故障演练

**① 目标说明**

模拟 gz-03 MySQL 主库不可用，将 gz-02 Slave1 提升为新主，验证 RTO < 5 分钟、RPO ≈ 0（半同步保障），固化切换 SOP，演练后恢复原拓扑。

**② 节点**：gz-02 / gz-03 / bj-01 / gz-01（ProxySQL）

**③ 切换步骤**

**步骤 6.1：模拟主库故障**

```bash
# 在 gz-03 执行（停止 MySQL 容器模拟宕机）
docker stop mysql
```

**步骤 6.2：gz-02 Slave1 提升为新主**

```bash
# 在 gz-02 执行
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
STOP REPLICA;
RESET REPLICA ALL;
SET GLOBAL read_only = OFF;
EOF
```

**步骤 6.3：更新 ProxySQL 写路由**

```bash
# 在 gz-03（或任意可访问 ProxySQL Admin 的节点）执行
# 若 gz-03 MySQL 已停止但 ProxySQL 容器仍在运行
docker exec -i proxysql mysql -h127.0.0.1 -P6032 -uadmin -padmin <<'EOF'
UPDATE mysql_servers SET hostname='100.79.132.125' WHERE hostgroup_id=0;
LOAD MYSQL SERVERS TO RUNTIME;
SAVE MYSQL SERVERS TO DISK;
EOF
```

**步骤 6.4：bj-01 Slave2 重新指向新主**

```bash
# 在 bj-01 执行
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
STOP REPLICA;
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='100.79.132.125',
  SOURCE_PORT=3306,
  SOURCE_USER='repl_user',
  SOURCE_PASSWORD='your_repl_password',
  SOURCE_AUTO_POSITION=1;
STOP REPLICA SQL_THREAD;
CHANGE REPLICATION SOURCE TO SOURCE_DELAY = 3600;
START REPLICA;
EOF
```

**步骤 6.5：记录 RTO**

记录从 Phase 6.1（停止 mysql 容器）到 Phase 6.3（ProxySQL 路由更新完成，业务恢复）的总耗时。目标：< 5 分钟。

**步骤 6.6：恢复原拓扑**

```bash
# gz-03 重启 MySQL Master
docker start mysql

# gz-02 重新设为从库（指向 gz-03）
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
SET GLOBAL read_only = ON;
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='100.92.5.116',
  SOURCE_PORT=3306,
  SOURCE_USER='repl_user',
  SOURCE_PASSWORD='your_repl_password',
  SOURCE_AUTO_POSITION=1;
START REPLICA;
EOF

# 还原 ProxySQL 写 hostgroup 指向 gz-03
docker exec -i proxysql mysql -h127.0.0.1 -P6032 -uadmin -padmin <<'EOF'
UPDATE mysql_servers SET hostname='100.92.5.116' WHERE hostgroup_id=0;
LOAD MYSQL SERVERS TO RUNTIME;
SAVE MYSQL SERVERS TO DISK;
EOF

# bj-01 重新指向 gz-03
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" <<'EOF'
STOP REPLICA;
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='100.92.5.116',
  SOURCE_PORT=3306,
  SOURCE_USER='repl_user',
  SOURCE_PASSWORD='your_repl_password',
  SOURCE_AUTO_POSITION=1;
STOP REPLICA SQL_THREAD;
CHANGE REPLICATION SOURCE TO SOURCE_DELAY = 3600;
START REPLICA;
EOF
```

**④ 预期输出**

切换过程中业务短暂中断（主库停止到 ProxySQL 路由更新完成期间），之后业务恢复正常。

**⑤ 验证方法**

```bash
# 切换完成后验证 gz-02 成为新主（演练阶段）
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "SHOW MASTER STATUS;"
# 期望：有 File/Position 输出，read_only=OFF

# 恢复原拓扑后验证 gz-03 重新为主
docker exec -i mysql mysql -uroot -p"${MYSQL_ROOT_PASSWORD}" -e "SHOW MASTER STATUS; SHOW STATUS LIKE 'rpl_semi_sync_source_clients';"
# 期望：semi_sync_source_clients = 2（gz-02 和 bj-01 均已接入）
```

**⑥ 回滚方式**

本 Phase 的"回滚"即步骤 6.6 恢复原拓扑，已包含在上述步骤中。
