# 操作手册：v1.0 → v1.1

## 演进概述

将监控栈（Prometheus + Grafana）从 bj-01 迁至 gz-01，并为全部四个节点部署标准 node-exporter，以实现监控链路与业务/运维节点解耦，消除"被监控节点宕机时监控同时不可用"的风险。

---

## 前置条件

| 项目 | 要求 |
|------|------|
| 当前架构版本 | V1.0（参见 `architecture/v1.0.md`） |
| gz-01 | Docker + Compose 已安装；`global_gateway` 网络已存在 |
| gz-03 / gz-02 | Docker + Compose 已安装；`global_gateway` 网络已存在 |
| bj-01 | Prometheus + Grafana 容器当前运行中；Docker + Compose 已安装 |
| Tailscale | 四节点互通，延迟 gz-01↔bj-01 约 35ms |
| Nginx（gz-01） | `monitor.jjmstart.com` 反代规则可修改 |

## 各 Phase 详细步骤

### Phase 1：停止 bj-01 监控服务

**① 目标说明**

停止 bj-01 上运行的 Prometheus 和 Grafana 容器。Grafana 数据（Dashboard、数据源配置）若需保留，先备份 volume 或导出 JSON。

**② 节点**：bj-01（100.118.69.78）

**③ 具体命令**

```bash
# 进入监控 compose 目录
cd /opt/docker/monitor

# 停止并移除 Prometheus 和 Grafana（保留 node-exporter）
docker compose stop prometheus grafana
docker compose rm -f prometheus grafana
```

如需备份 Grafana Dashboard：

```bash
# 导出所有 Dashboard（Grafana HTTP API）
curl -s http://100.118.69.78:3000/api/search?type=dash-db \
  -u "admin:$GRAFANA_ADMIN_PASSWORD" | jq '.[].uid' -r | while read uid; do
    curl -s http://100.118.69.78:3000/api/dashboards/uid/$uid \
      -u "admin:$GRAFANA_ADMIN_PASSWORD" > dashboard_$uid.json
done
```

**④ 预期输出**

```
Stopping prometheus ... done
Stopping grafana    ... done
Removing prometheus ... done
Removing grafana    ... done
```

**⑤ 验证方法**

```bash
docker compose ps
# 应只剩 node-exporter 和其他服务，无 prometheus/grafana
```

**⑥ 回滚方式**

```bash
cd /opt/docker/monitor && docker compose up -d prometheus grafana
```

### Phase 2：gz-01 部署监控全栈

**① 目标说明**

在 gz-01 创建 `/opt/docker/monitor/` 目录结构，编写 Prometheus + Grafana + node-exporter 的 compose 文件与采集配置，启动服务。

**② 节点**：gz-01（100.117.7.75）

**③ 具体命令**

创建目录结构：

```bash
mkdir -p /opt/docker/monitor/prometheus
```

创建 `/opt/docker/monitor/docker-compose.yml`（含 prometheus / grafana / node-exporter 三个服务，连接 `global_gateway` 网络，prometheus 绑定容器内 9090，grafana 绑定 `100.117.7.75:3000`，node-exporter 绑定 Docker 内网 9100）。

创建 `/opt/docker/monitor/prometheus/prometheus.yml`，配置四节点 targets：

```yaml
scrape_configs:
  - job_name: 'node'
    static_configs:
      - targets:
          - 'node-exporter:9100'          # gz-01 本机（Docker 服务名）
          - '100.79.132.125:9100'         # gz-02
          - '100.92.5.116:9100'           # gz-03
          - '100.118.69.78:9100'          # bj-01
```

启动服务：

```bash
cd /opt/docker/monitor
docker compose up -d
```

**④ 预期输出**

```
Creating network "monitor_default" with the default driver
Creating prometheus    ... done
Creating grafana       ... done
Creating node-exporter ... done
```

**⑤ 验证方法**

```bash
# 检查容器状态
docker compose ps

# 检查 Prometheus 目标状态
curl http://localhost:9090/api/v1/targets | python3 -m json.tool | grep -E '"health"|"job"'
# 期望：gz-01 node-exporter 状态 "up"（gz-02/03/bj-01 node-exporter 尚未部署，状态 "down"，Phase 3-5 后全部变为 "up"）

# 访问 Grafana
curl -I http://100.117.7.75:3000
# 期望：HTTP/1.1 302 Found（跳转登录页）
```

**⑥ 回滚方式**

```bash
cd /opt/docker/monitor && docker compose down
```

### Phase 6：更新 Nginx 反代

**① 目标说明**

将 gz-01 Nginx 中 `monitor.jjmstart.com` 的 `proxy_pass` 由 `http://100.118.69.78:3000` 改为本地 `http://grafana:3000`（Docker 服务名）或 `http://100.117.7.75:3000`，消除跨城 ~35ms 延迟。

**② 节点**：gz-01（100.117.7.75）

**③ 具体命令**

```bash
# 编辑 Nginx 配置
vi /opt/docker/nginx/conf.d/monitor.conf

# 修改 proxy_pass：
# 原：proxy_pass http://100.118.69.78:3000;
# 改：proxy_pass http://grafana:3000;
#（grafana 容器与 nginx 容器同在 global_gateway 网络，可用服务名）

# 重载 Nginx（无停机）
docker exec nginx nginx -s reload
```

**④ 预期输出**

```
# nginx -s reload 无输出（成功）
```

**⑤ 验证方法**

```bash
# 访问 monitor 域名（需本地 hosts 或公网 DNS）
curl -I https://monitor.jjmstart.com:8443
# 期望：HTTP/2 200 或 302 重定向至 /login

# 检查 Nginx 访问日志，确认无 502/504
docker logs nginx --tail 20
```

**⑥ 回滚方式**

```bash
# 改回 proxy_pass http://100.118.69.78:3000;
docker exec nginx nginx -s reload
```

### Phase 7：验收

**① 目标说明**

确认四节点 node-exporter 数据全部被 Prometheus 采集，Grafana 面板可正常访问，监控链路完全切换至 gz-01。

**② 节点**：gz-01（100.117.7.75）

**③ 验证命令**

```bash
# 检查所有 targets 状态
curl -s 'http://localhost:9090/api/v1/targets' | python3 -c "
import json,sys
data = json.load(sys.stdin)
for t in data['data']['activeTargets']:
    print(t['labels']['instance'], t['health'])
"
# 期望：四个 :9100 实例全部 health=up

# 检查基础指标可查询
curl -s 'http://localhost:9090/api/v1/query?query=node_load1' | python3 -m json.tool
# 期望：四个节点均有 node_load1 数据
```
