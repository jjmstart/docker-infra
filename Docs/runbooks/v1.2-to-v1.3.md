# 操作手册：v1.2 → v1.3

## 演进概述

将 `/opt/docker` 整体纳入 Git 版本控制并推送至 GitHub Private 仓库，为所有服务的 `.env` 文件建立 `.env.example` 模板，并在四个节点上完成仓库 clone，实现配置即代码的基础管理层——集群服务本身不做任何变更。

## 各 Phase 详细步骤

### Phase 1：GitHub 准备

**① 目标说明**

在 GitHub 创建 Private 仓库，在 gz-03 生成 SSH 密钥并添加到 GitHub，确保 gz-03 可以通过 SSH 推送代码。

**② 节点**：gz-03（100.92.5.116）+ GitHub 网页操作

**③ 具体命令**

步骤 1.1：在 gz-03 生成 SSH 密钥（如已有可跳过）

```bash
# 检查是否已有 SSH 密钥
ls ~/.ssh/id_ed25519.pub 2>/dev/null && echo "已有密钥" || echo "需要生成"

# 如需生成（邮箱替换为你的 GitHub 注册邮箱）
ssh-keygen -t ed25519 -C "your_email@example.com"
# 一路回车使用默认路径和空密码（简单起见）

# 查看公钥内容（下一步要复制到 GitHub）
cat ~/.ssh/id_ed25519.pub
```

步骤 1.2：GitHub 网页操作

1. 登录 GitHub → 右上角头像 → **Settings**
2. 左栏 **SSH and GPG keys** → **New SSH key**
3. Title 填 `gz-03`，Key 粘贴上一步输出的公钥内容 → **Add SSH key**
4. 回到 GitHub 首页 → **New repository**
5. Repository name 填 `docker-infra`（或自定义），选 **Private**，**不要**勾选 Initialize README → **Create repository**
6. 记录仓库 SSH 地址，格式为 `git@github.com:your-username/docker-infra.git`

步骤 1.3：验证 SSH 连通性

```bash
ssh -T git@github.com
```

**④ 预期输出**

```
Hi your-username! You've successfully authenticated, but GitHub does not provide shell access.
```

**⑤ 验证方法**

输出包含你的 GitHub 用户名即为成功。

**⑥ 回滚方式**

本 Phase 纯准备工作，不影响任何现有服务，无需回滚。

### Phase 2：Git 初始化与 .gitignore

**① 目标说明**

在 `/opt/docker` 目录初始化 Git 仓库，创建 `.gitignore` 文件，确保数据目录、真实 `.env` 文件、SQL 备份等敏感/大文件被排除在版本控制之外。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

```bash
cd /opt/docker
git init
git branch -M main
```

创建 `/opt/docker/.gitignore`，内容如下：

```gitignore
# ========= 数据目录（绝对不进 Git）=========
backend/mysql/data/
backend/redis/data/
backend/proxysql/data/

# ========= 真实密码文件（不进 Git，只有 .env.example 进入）=========
**/.env

# ========= SQL 备份文件 =========
backend/mysql/*.sql
backend/mysql/*.sql.gz

# ========= SSL 证书 =========
nginx/certs/
**/*.pem
**/*.key

# ========= 系统文件 =========
.DS_Store
*.log
```

验证排除效果（非常重要）：

```bash
# 预演 git add，查看哪些文件会被追踪
git add --dry-run . 2>&1 | head -50
```

仔细检查输出，确认以下目录/文件**不出现**：
- `backend/mysql/data/`
- `backend/redis/data/`
- 任何 `.env` 文件
- 任何 `.sql` 文件

**④ 预期输出**（`git add --dry-run` 应看到的类型）

```
add 'Docs/README.md'
add 'Docs/architecture/v1.2.md'
add 'backend/mysql/docker-compose.yml'
add 'backend/proxysql/proxysql.cnf'
...（配置文件和文档，无 data/ 目录，无 .env，无 .sql）
```

**⑤ 验证方法**

```bash
# 确认 mysql/data 被排除
git check-ignore -v backend/mysql/data/
# 期望输出：.gitignore:3:backend/mysql/data/    backend/mysql/data/

# 确认 .env 被排除（如果存在的话）
git check-ignore -v backend/mysql/.env
# 期望输出：.gitignore:8:**/.env    backend/mysql/.env
```

**⑥ 回滚方式**

```bash
# 如需撤销 git init（删除 .git 目录，仓库消失，文件不受影响）
rm -rf /opt/docker/.git
```

### Phase 3：创建 .env.example 模板

**① 目标说明**

为四个服务的 `.env` 文件各创建一份 `.env.example`，内容保留变量名但将真实密码替换为占位符，使新节点部署时知道需要填写哪些变量。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

先查看各 `.env` 的变量名（内容不会出现在 Git 中）：

```bash
# 查看各 .env 包含哪些变量名（只看 KEY 不看 VALUE）
grep -oP '^[A-Z_]+(?==)' backend/mysql/.env
grep -oP '^[A-Z_]+(?==)' backend/proxysql/.env
grep -oP '^[A-Z_]+(?==)' backend/ruoyi/.env
grep -oP '^[A-Z_]+(?==)' backend/redis/.env
```

根据输出，手动为每个服务创建 `.env.example`。以 MySQL 为例：

```bash
# 创建 backend/mysql/.env.example
cat > backend/mysql/.env.example << 'EOF'
# MySQL 根密码
MYSQL_ROOT_PASSWORD=your_root_password_here

# 数据库名
MYSQL_DATABASE=your_database_name

# 复制账号密码（各节点 .env 中必须相同）
MYSQL_REPL_PASSWORD=your_repl_password_here
EOF
```

ProxySQL（`backend/proxysql/.env.example`）、若依（`backend/ruoyi/.env.example`）、Redis（`backend/redis/.env.example`）参照相同方式创建，占位符统一用 `your_xxx_here` 格式。

**④ 预期输出**

```bash
ls backend/mysql/.env.example backend/proxysql/.env.example \
   backend/ruoyi/.env.example backend/redis/.env.example
# 四个文件均存在
```

**⑤ 验证方法**

```bash
# 确认 .env.example 能被 Git 追踪（不被 .gitignore 排除）
git check-ignore -v backend/mysql/.env.example
# 期望：无输出（即没有被排除，会进入 Git）

# 确认 .env 仍被排除
git check-ignore -v backend/mysql/.env
# 期望：输出 .gitignore 规则
```

**⑥ 回滚方式**

```bash
rm backend/mysql/.env.example backend/proxysql/.env.example \
   backend/ruoyi/.env.example backend/redis/.env.example
```

### Phase 4：初始 commit 与推送

**① 目标说明**

将所有配置文件、文档、`.env.example` 模板、`.gitignore` 打包为第一个 commit，推送至 GitHub。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

配置 Git 用户信息（首次使用必须）：

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

添加远程仓库（替换为你的实际仓库地址）：

```bash
git remote add origin git@github.com:your-username/docker-infra.git
```

暂存所有文件并检查：

```bash
git add .

# 最后检查一遍即将提交的内容
git status
```

**再次确认** status 输出中没有 `data/`、`.env`、`.sql` 等文件，然后提交：

```bash
git commit -m "feat: initial infrastructure as code

- Docker Compose configurations for all 4 nodes (gz-01/02/03/bj-01)
- MySQL 1-master 2-slave + ProxySQL read/write splitting (V1.2 state)
- Prometheus/Grafana monitoring stack on gz-01
- Docs with architecture snapshots, runbooks, and retrospectives
- .gitignore to exclude data dirs, .env files, and SQL backups
- .env.example templates for all services"

# 推送
git push -u origin main
```

**④ 预期输出**

```
Enumerating objects: XX, done.
Counting objects: 100% (XX/XX), done.
...
To git@github.com:your-username/docker-infra.git
 * [new branch]      main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.
```

**⑤ 验证方法**

```bash
# 本地验证
git log --oneline
# 期望：出现刚才的 commit

# 在浏览器打开 GitHub 仓库，确认文件已上传
# 检查仓库中是否存在：
#   ✓ backend/mysql/docker-compose.yml
#   ✓ backend/mysql/.env.example
#   ✗ backend/mysql/.env（不应存在）
#   ✗ backend/mysql/data/（不应存在）
```

**⑥ 回滚方式**

```bash
# 撤销最后一次本地 commit（保留文件，仅撤销 commit）
git reset --soft HEAD~1

# 如已推送到 GitHub，需要强制推送回退（谨慎）
# git push --force origin main
```

### Phase 5：打历史里程碑 tag

**① 目标说明**

为当前 commit（即 V1.2 的集群状态）打上 `arch-v1.2` tag，作为 Git 历史基线。后续每完成一次架构演进，打一个对应 tag，替代手动维护"历史归档"文件的方式。

**② 节点**：gz-03（100.92.5.116）

**③ 具体命令**

```bash
# 创建带注释的 tag（annotated tag，推荐，包含元信息）
git tag -a arch-v1.2 -m "MySQL HA: 1-master 2-slave + semi-sync + GTID + ProxySQL read/write splitting"

# 推送 tag 到 GitHub（git push 默认不推送 tag）
git push origin arch-v1.2
```

**④ 预期输出**

```
To git@github.com:your-username/docker-infra.git
 * [new tag]         arch-v1.2 -> arch-v1.2
```

**⑤ 验证方法**

```bash
git tag -l
# 期望：输出 arch-v1.2

# 验证 tag 内容
git show arch-v1.2 --stat
# 期望：显示 tag 注释和对应 commit 的文件统计

# 可选：验证 tag 可以查看历史文件
git show arch-v1.2:Docs/architecture/v1.2.md | head -5
# 期望：输出 v1.2.md 的前几行内容
```

**⑥ 回滚方式**

```bash
# 删除本地 tag
git tag -d arch-v1.2

# 删除远程 tag
git push origin --delete arch-v1.2
```

### Phase 6：其他三节点 clone 仓库

**① 目标说明**

在 gz-01、gz-02、bj-01 上各自生成 SSH 密钥并添加到 GitHub，clone 仓库到 `/opt/docker`，验证目录结构与 gz-03 一致。

**② 节点**：gz-01 / gz-02 / bj-01（分别操作，步骤相同）

> **注意**：各节点原本就有 `/opt/docker` 目录，需要先将原有内容替换为 Git 仓库。操作前确认各节点没有未备份的本地修改。

**③ 具体命令**（以 gz-01 为例，gz-02 / bj-01 步骤相同）

步骤 6.1：生成 SSH 密钥并添加到 GitHub

```bash
# 在 gz-01 执行
ssh-keygen -t ed25519 -C "gz-01@docker-infra"
cat ~/.ssh/id_ed25519.pub
```

将输出的公钥添加到 GitHub：Settings → SSH and GPG keys → New SSH key，Title 填 `gz-01`。

```bash
# 验证连通性
ssh -T git@github.com
# 期望：Hi your-username! You've successfully authenticated...
```

步骤 6.2：备份并替换原有 `/opt/docker` 目录

```bash
# 备份原有目录（重要）
cp -r /opt/docker /opt/docker.bak.$(date +%Y%m%d)

# 删除原有目录（Git clone 要求目标目录不存在或为空）
# 方式 A：删除原目录，重新 clone
rm -rf /opt/docker
git clone git@github.com:your-username/docker-infra.git /opt/docker

# 方式 B：如不想删除目录（比如有本地日志等），在空临时目录 clone 后覆盖
# git clone git@github.com:your-username/docker-infra.git /tmp/docker-infra
# cp -r /tmp/docker-infra/. /opt/docker/
# cd /opt/docker && git init --separate-git-dir（较复杂，推荐方式 A）
```

步骤 6.3：恢复本节点的 `.env` 文件

clone 完成后，仓库中没有 `.env` 文件（被 `.gitignore` 排除），需要根据 `.env.example` 手动创建：

```bash
# 以 gz-01 为例（该节点只运行 monitor，可能只需要 monitor 相关配置）
# 参考 .env.example 创建 .env
cd /opt/docker
cat backend/mysql/.env.example   # 查看需要填哪些变量
# 然后用 vi 或 echo 创建真实 .env
```

**④ 预期输出**

```
Cloning into '/opt/docker'...
remote: Enumerating objects: XX, done.
...
Receiving objects: 100% (XX/XX), XX KiB | XX MiB/s, done.
```

**⑤ 验证方法**

```bash
cd /opt/docker

# 确认是 Git 仓库
git status
# 期望：On branch main, nothing to commit

# 确认 remote 正确
git remote -v
# 期望：origin  git@github.com:your-username/docker-infra.git (fetch/push)

# 确认文档存在
ls Docs/architecture/
# 期望：README.md  v1.0.md  v1.1.md  v1.2.md  v1.3.md
```

对 gz-02、bj-01 重复以上步骤。

**⑥ 回滚方式**

```bash
# 恢复备份
rm -rf /opt/docker
mv /opt/docker.bak.$(date +%Y%m%d) /opt/docker
```

```bash

cd /opt/docker
git add backend/proxysql/proxysql.cnf
git commit -m "fix: proxysql read hostgroup weight adjusted"
git push

# 其他节点同步
ssh gz-01 "cd /opt/docker && git pull"
ssh gz-02 "cd /opt/docker && git pull"

# 完成新版本架构演进后，打 tag

git push origin arch-v1.3
```
