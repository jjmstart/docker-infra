# 操作手册：v1.7 → v1.8

## 演进概述

在 V1.7 备份恢复闭环的基础上，将两台新购广州节点（gz-04 / gz-05）安全、可重复地接入现有基础设施：通过新增 `roles/tailscale/` 实现网络层 IaC，扩展 `roles/docker-daemon/` 补全 Docker Engine 安装与 `global_gateway` 网络创建，改造 `roles/node-exporter/` 支持纯节点采集（无 mysqld-exporter），并将 gz-04 / gz-05 纳入 Prometheus 监控面板。新节点 V1.8 阶段不承载任何业务服务，只完成基础接入。

**子任务 1.4：新建 roles/tailscale/ role**

**③ 具体命令**

```bash
cd /opt/docker-infra

# 创建 role 目录结构
mkdir -p roles/tailscale/{tasks,handlers}
```

创建 `roles/tailscale/tasks/main.yml`，内容如下：

```yaml
---
- name: Install Tailscale apt signing key
  ansible.builtin.apt_key:
    url: "https://pkgs.tailscale.com/stable/ubuntu/{{ ansible_distribution_release }}.gpg"
    state: present

- name: Add Tailscale apt repository
  ansible.builtin.apt_repository:
    repo: "deb https://pkgs.tailscale.com/stable/ubuntu {{ ansible_distribution_release }} main"
    state: present
    update_cache: true

- name: Install tailscale package
  ansible.builtin.apt:
    name: tailscale
    state: present

- name: Enable and start tailscaled service
  ansible.builtin.systemd:
    name: tailscaled
    state: started
    enabled: true

- name: Check current Tailscale backend state
  ansible.builtin.command: tailscale status --json
  register: ts_status_raw
  changed_when: false
  failed_when: false

- name: Join tailnet with auth key
  ansible.builtin.command: >
    tailscale up
    --authkey={{ tailscale_authkey }}
    --hostname={{ inventory_hostname }}
  when: >
    ts_status_raw.rc != 0 or
    (ts_status_raw.stdout | from_json).BackendState != 'Running'
  no_log: true
  changed_when: true
```

创建 `roles/tailscale/handlers/main.yml`，内容如下：

```yaml

# 本 role 暂无 handler
```

**子任务 1.6：扩展 roles/docker-daemon/ 安装 Docker Engine**

当前 `roles/docker-daemon/tasks/main.yml` 只配置 pip、SDK 和 `daemon.json`，未安装 Docker Engine。在现有内容**之前**插入以下 Docker Engine 安装 task：

**③ 具体命令**

将 `roles/docker-daemon/tasks/main.yml` 改为如下完整内容（在现有 pip/SDK 相关 task 之前插入 Docker Engine 安装部分）：

```yaml
---
# ── Docker Engine 安装 ─────────────────────────────────────────────────────────
- name: Install Docker Engine prerequisites
  ansible.builtin.apt:
    name:
      - ca-certificates
      - curl
      - gnupg
    state: present
    update_cache: true

- name: Create /etc/apt/keyrings directory
  ansible.builtin.file:
    path: /etc/apt/keyrings
    state: directory
    mode: "0755"

- name: Add Docker apt signing key
  ansible.builtin.get_url:
    url: "https://download.docker.com/linux/ubuntu/gpg"
    dest: /etc/apt/keyrings/docker.asc
    mode: "0644"

- name: Add Docker apt repository
  ansible.builtin.apt_repository:
    repo: >-
      deb [arch=amd64 signed-by=/etc/apt/keyrings/docker.asc]
      https://download.docker.com/linux/ubuntu
      {{ ansible_distribution_release }} stable
    state: present
    update_cache: true
    filename: docker

- name: Install Docker Engine packages
  ansible.builtin.apt:
    name:
      - docker-ce
      - docker-ce-cli
      - containerd.io
      - docker-buildx-plugin
      - docker-compose-plugin
    state: present

- name: Enable and start docker service
  ansible.builtin.systemd:
    name: docker
    state: started
    enabled: true

- name: Create global_gateway docker network
  ansible.builtin.command: docker network create global_gateway
  register: network_result
  changed_when: network_result.rc == 0
  failed_when: network_result.rc != 0 and 'already exists' not in network_result.stderr

# ── Python Docker SDK 与 pip 配置 ──────────────────────────────────────────────
- name: Remove outdated python3-docker apt package
  ansible.builtin.apt:
    name: python3-docker
    state: absent

- name: Ensure python3-pip is installed
  ansible.builtin.apt:
    name: python3-pip
    state: present
    update_cache: false

- name: Configure pip mirror
  ansible.builtin.copy:
    dest: /etc/pip.conf
    mode: "0644"
    content: |
      [global]
      index-url = https://mirrors.aliyun.com/pypi/simple/
      trusted-host = mirrors.aliyun.com

- name: Remove root user-level pip config with stale mirror
  ansible.builtin.file:
    path: "{{ item }}"
    state: absent
  loop:
    - /root/.pip/pip.conf
    - /root/.config/pip/pip.conf

- name: Install Python Docker SDK via pip
  ansible.builtin.pip:
    name: docker
    state: latest
    extra_args: "{{ '--break-system-packages' if ansible_python_version is version('3.11', '>=') else '' }}"

# ── Docker daemon 配置 ─────────────────────────────────────────────────────────
- name: Ensure /etc/docker directory exists
  ansible.builtin.file:
    path: /etc/docker
    state: directory
    mode: "0755"

- name: Configure Docker daemon
  ansible.builtin.template:
    src: daemon.json.j2
    dest: /etc/docker/daemon.json
    mode: "0644"
  notify: reload docker daemon
```

**④ 预期输出**：文件保存，YAML 无报错。

**⑤ 验证方法**

```bash
python3 -c "import yaml; yaml.safe_load(open('roles/docker-daemon/tasks/main.yml'))" && echo "YAML OK"
```

**⑥ 回滚方式**：`git checkout -- roles/docker-daemon/tasks/main.yml` 恢复原内容。

> **注意**：现有四台节点已手动安装 Docker Engine，重复执行此 role 时 apt 模块会因 `state: present` 跳过安装步骤（`changed=0`），`docker network create` task 会因 `already exists` 提前返回（`changed=0`），整体幂等。

**子任务 1.7：改造 roles/node-exporter/ 支持纯采集节点**

当前 `roles/node-exporter/templates/docker-compose.yml.j2` 始终同时渲染 `node-exporter` 和 `mysqld-exporter` 两个服务。gz-04/gz-05 是空白节点，无 MySQL，不能启动 mysqld-exporter。

**③ 具体命令**

将 `roles/node-exporter/templates/docker-compose.yml.j2` 改为以下内容（新增 Jinja2 条件块）：

```yaml
services:
  node-exporter:
    image: prom/node-exporter:v1.8.0
    container_name: node-exporter
    uts: host
    restart: unless-stopped

    ports:
      - "{{ ansible_host }}:{{ node_exporter_port }}:9100"

    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro

    command:
      - '--path.procfs=/host/proc'
      - '--path.sysfs=/host/sys'
      - '--path.rootfs=/rootfs'

    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

{% if inventory_hostname in groups['mysqld_exporter_nodes'] %}
  mysqld-exporter:
    image: prom/mysqld-exporter:v0.15.1
    container_name: mysqld-exporter
    restart: unless-stopped

    volumes:
      - ./mysqld-exporter.cnf:/etc/mysqld-exporter.cnf:ro

    ports:
      - "{{ ansible_host }}:{{ mysqld_exporter_port }}:9104"

    command:
      - --config.my-cnf=/etc/mysqld-exporter.cnf
      - --collect.slave_status
      - --collect.slave_hosts
      - --collect.global_status
      - --collect.global_variables
      - --collect.info_schema.processlist

    networks:
      - {{ docker_network_name }}

    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

networks:
  {{ docker_network_name }}:
    external: true
{% endif %}
```

## 验收标准

| 验收项 | 可验证的成功条件 |
|--------|-----------------|
| Tailscale 接入 | gz-04 / gz-05 出现在 Tailscale 控制台且状态为 Connected；`tailscale ping gz-04` / `gz-05` 从 bj-01 均可达 |

| Prometheus targets | Prometheus UI 中 `up{instance="<gz-04-tailscale-ip>:9100"} = 1` 及 gz-05 同 |
| Grafana 可见 | Node Exporter 面板 Host 列表中新节点主机指标（CPU / 内存 / 磁盘）可见 |

| 敏感信息 | `vault/secrets.yml` 保持 AES256 加密；inventory / roles / playbooks 中无明文密码、authkey |
| Auth Key | Tailscale auth key 在验收完成后已 revoke |
