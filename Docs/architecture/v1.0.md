# 架构快照 v1.0

## 文档说明

V1.0 为集群初始拓扑基线：四节点（gz-01 入口网关 / gz-03 业务主 / gz-02 业务副 / bj-01 运维灾备）+ Redis 哨兵集群，监控栈（Prometheus/Grafana）位于 bj-01。**本版本为历史归档，非当前最新版本，请参阅 [v1.2.md](v1.2.md)。**

---

## AI 上下文引导（Context Bootstrap）

> 本节供 AI 快速建立上下文，人工阅读可跳过。

**仓库根目录与管理方式**

- 仓库根目录：`/opt/docker`
- 所有服务均以 Docker Compose 管理
- 网络：`global_gateway`（各节点已创建）

**节点互联方式**

所有节点通过 **Tailscale WireGuard** 加密隧道互联，不依赖公网端口暴露。

**关键文件路径索引**

| 文件路径 | 用途 |
|----------|------|
| `/opt/docker/nginx/docker-compose.yml` | gz-01 Nginx 入口网关 |
| `/opt/docker/backend/mysql/docker-compose.yml` | gz-03 MySQL 主库（单点） |
| `/opt/docker/backend/redis/docker-compose.yml` | gz-03 Redis 主 + Sentinel1 |
| `/opt/docker/monitor/docker-compose.yml` | bj-01 Prometheus + Grafana |
| `/opt/docker/jenkins/docker-compose.yml` | bj-01 Jenkins CI/CD |

**本版本核心技术决策**

| 决策点 | 选型 | 理由 |
|--------|------|------|
| 入口层 | 单节点 Nginx（gz-01） | 统一 SSL 卸载与反代，简化配置 |
| 数据库 | MySQL 单点主库（gz-03） | 初始规模，暂不引入主从复杂度 |
| 缓存高可用 | Redis 哨兵三节点 | 跨城三节点 quorum=2，故障自动切换 |
| 监控 | Prometheus + Grafana（bj-01） | 与业务节点同城，复用 bj-01 内存 |
| 节点互联 | Tailscale WireGuard | 免公网暴露，端到端加密 |

## 架构拓扑图

```
互联网用户
    │
    ▼
gz-01（入口网关）
├── Nginx（反向代理 + SSL 卸载 + 静态主站）
│     ├── jjmstart.com         → 本地静态文件 (0ms)
│     ├── ruoyi.jjmstart.com   → gz-03 + gz-02 负载均衡 (同城 ~5ms)
│     ├── diary.jjmstart.com   → bj-01 (跨城 ~35ms)
│     ├── jenkins.jjmstart.com → bj-01 (跨城 ~35ms)
│     └── monitor.jjmstart.com → bj-01 (跨城 ~35ms)
│
├────────── 同城 ~5ms ──────────┐
│                                │
gz-03（业务主节点）               gz-02（业务副节点）
├── ruoyi-admin-1                ├── ruoyi-admin-2
├── MySQL 主库                   ├── Redis Slave1
├── Redis Master                 └── Sentinel2
└── Sentinel1
                    ↕ Tailscale ~35ms
             bj-01（运维 + 灾备）
             ├── Jenkins CI/CD
             ├── Prometheus + Grafana
             ├── Diary (Memos)
             ├── Redis Slave2（异地灾备）
             └── Sentinel3
```

Redis 哨兵集群：

```
gz-03 Master ──复制──→ gz-02 Slave1（同城，priority=100）
     │
     └─────复制──→ bj-01 Slave2（异地，priority=200）

Sentinel 集群：3 节点（gz-03 / gz-02 / bj-01），quorum=2
故障切换顺序：gz-02(同城) > bj-01(异地)
```

## 网络互联

所有节点通过 Tailscale WireGuard 加密隧道互联，不依赖公网端口暴露。

| 链路 | 延迟 | 用途 |
|------|------|------|
| gz-01 ↔ gz-03 | ~5ms | Nginx → ruoyi-admin-1 |
| gz-01 ↔ gz-02 | ~5ms | Nginx → ruoyi-admin-2 |
| gz-01 ↔ bj-01 | ~35ms | Nginx → Jenkins / Grafana / Diary |
| gz-03 ↔ gz-02 | ~5ms | MySQL 跨节点访问、Redis 同城复制 |
| gz-03 ↔ bj-01 | ~35ms | Redis 异地复制 |
