# 架构快照 v1.3

## 文档说明

V1.3 相对 V1.2 的核心变更：**将 `/opt/docker` 整体纳入 Git 版本控制，推送至 GitHub Private 仓库，并为所有服务的 `.env` 文件建立 `.env.example` 模板**，实现配置即代码（Infrastructure as Code）的基础管理层。集群服务拓扑与 V1.2 完全相同，变更仅在管理层。**本版本为历史归档，非当前最新版本，请参阅 [v1.4.md](v1.4.md)。**

---

## AI 上下文引导（Context Bootstrap）

> 本节供 AI 快速建立上下文，人工阅读可跳过。

**仓库根目录与管理方式**

- 仓库根目录：`/opt/docker`
- **V1.3 起：`/opt/docker` 已是 Git 仓库**，远程为 GitHub Private 仓库（`origin`）
- 所有服务均以 Docker Compose 管理，网络为 `global_gateway`
- 节点 hostname 约定：`gz-01` / `gz-02` / `gz-03` / `bj-01`
- **各节点均已 clone 同一仓库**，日常通过 `git pull` 同步配置变更
- **敏感信息管理**：`.env` 文件在 `.gitignore` 中排除，仓库中只保留 `.env.example` 模板；真实 `.env` 通过线下渠道在各节点手动维护

**节点互联方式**

所有节点通过 **Tailscale WireGuard** 加密隧道互联，不依赖公网端口暴露。

**关键文件路径索引**


**本版本核心技术决策**

| 决策点 | 选型 | 理由 |
|--------|------|------|
| 版本控制工具 | **Git + GitHub Private** | 标准化变更追踪；配置与文档变更捆绑在同一 commit；Private 仓库防止配置信息公开 |
| 敏感信息处理 | **`.env.example` 模板 + 手动维护 `.env`** | 当前单人运维规模，操作最简单；需要时可升级为 SOPS 加密方案 |
| 历史版本标记 | **Git tag**（如 `arch-v1.2`） | 替代手动维护"历史归档"文件；`git show arch-v1.2:Docs/architecture/v1.2.md` 即可查看任意历史快照 |
| 多节点同步 | **各节点独立 `git pull`** | 无需引入 Ansible 等额外工具；每台节点只运行自身相关 compose，不受其他节点配置影响 |

## 与上一版本的差异（相对 v1.2）

- **`/opt/docker` 纳入 Git 管理**：初始化 Git 仓库，推送至 GitHub Private 仓库；配置变更与文档变更从此捆绑在同一 commit，解决了"文档与配置分叉"的问题。
- **敏感信息治理**：`.env` 文件加入 `.gitignore`，为四个服务（mysql / proxysql / ruoyi / redis）各新增 `.env.example` 模板，明确密码变量名而不暴露真实值；`mysql/data/`、`*.sql` 等数据目录与备份文件同步排除。
- **各节点统一 `git pull` 同步工作流**：gz-01 / gz-02 / bj-01 均 clone 同一仓库，日常配置变更通过 `git pull` 分发，替代此前各节点手动单独维护的方式。
